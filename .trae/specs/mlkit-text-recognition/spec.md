
# ML Kit 文本识别 - 产品需求文档

## 概述

- **总结**: 在 AutoBot RPA 应用中集成 Google ML Kit 文本识别功能，实现从屏幕截图中检测、识别文本，并通过脚本动作利用这些文本进行自动化操作。
- **目的**: 增强自动化能力，使应用能够识别屏幕上的文本内容，从而实现更智能的自动化流程。
- **目标用户**: AutoBot RPA 的用户，需要通过文本识别来定位或交互屏幕元素。

## 目标

- 集成 Google ML Kit 文本识别 SDK 到项目中
- 在 ScriptAction 中添加新的文本识别相关动作类型
- 实现文本识别服务，提供文本检测、文本搜索、文本定位等功能
- 在 UI 中支持配置和编辑新的文本识别动作
- 在自动化引擎中实现文本识别动作的执行逻辑

## 非目标（范围外）

- 不实现离线语言模型的动态下载（使用设备默认语言）
- 不实现手写体文本识别
- 不实现实时摄像头文本识别（只处理截图）

## 背景与上下文

AutoBot RPA 是一个 Android 自动化应用，目前支持坐标点击、图像识别等功能。文本识别是自动化流程中重要的一环，能够通过文本内容来定位元素，提高自动化的稳定性和适应性。

Google ML Kit 提供了免费且高性能的设备端文本识别 API，适合集成到本项目中。

## 功能需求

### FR-1: 集成 ML Kit 依赖
- 在 app/build.gradle 中添加 ML Kit 文本识别 SDK 依赖
- 配置必要的权限和资源

### FR-2: 创建文本识别服务
- 创建 TextRecognitionService 单例类
- 实现初始化、图像预处理、文本检测功能
- 提供文本搜索和定位的接口
- 支持获取文本块的坐标信息

### FR-3: 扩展 ScriptAction 数据模型
- 添加 `FindText` 动作类型，用于查找屏幕上的特定文本并获取其位置
- 扩展 `ConditionType` 枚举，添加 `TEXT_FOUND` 和 `TEXT_NOT_FOUND` 条件类型
- 更新 ScriptConverters 以支持新类型的序列化/反序列化

### FR-4: 更新自动化引擎
- 在 AutomationEngine 中实现 `findText` 动作的执行逻辑
- 实现文本条件判断的逻辑
- 支持将文本识别结果保存到变量系统中

### FR-5: 更新 UI 界面
- 在 AddActionDialog 中添加新的文本识别相关动作选项
- 创建 EditFindTextDialog 用于配置 FindText 动作
- 更新 EditConditionDialog 支持文本条件配置
- 更新 ActionItemCard 以显示新动作的详情

## 非功能需求

### NFR-1: 性能要求
- 文本识别应在 500ms 内完成（中等配置设备）
- 支持超时配置（默认 5000ms）

### NFR-2: 兼容性要求
- 支持 Android 8.0 (API 26) 及以上版本
- 保持现有功能的完整性

### NFR-3: 错误处理
- 文本识别失败应有友好的错误提示
- 处理图像无法识别等边界情况

## 约束

- **技术**: 必须使用 Google ML Kit 文本识别 SDK（设备端）
- **业务**: 保持与现有架构的一致性（单例服务、协程、Hilt DI）
- **依赖**: 依赖现有 ScreenshotManager 提供截图功能

## 假设

- 用户设备上已安装或可以使用 Google Play Services（ML Kit 依赖）
- 设备有足够的内存和处理能力进行文本识别
- 文本主要为中文或英文（ML Kit 默认支持多语言）

## 接受标准

### AC-1: FindText 动作正常工作
- **给定**: 屏幕上有可识别的文本
- **当**: 执行 FindText 动作，传入目标文本和超时配置
- **则**: 应用能成功找到文本并返回其中心坐标，相似度高于阈值时保存到变量（如果配置了）
- **验证方式**: programmatic

### AC-2: 条件判断支持文本检测
- **给定**: 条件类型为 TEXT_FOUND 或 TEXT_NOT_FOUND
- **当**: 脚本执行到条件判断时
- **则**: 正确判断屏幕上是否存在指定文本，并执行相应分支
- **验证方式**: programmatic

### AC-3: UI 配置界面完整
- **给定**: 用户在编辑脚本
- **当**: 选择添加 FindText 动作或文本条件
- **则**: 显示完整的配置界面，包括目标文本、超时、阈值、调试模式等选项
- **验证方式**: human-judgment

### AC-4: 错误处理友好
- **给定**: 文本识别过程中出现错误（如无截图权限、识别失败）
- **当**: 执行 FindText 动作
- **则**: 记录清晰的错误日志，不导致应用崩溃
- **验证方式**: programmatic

### AC-5: 与现有功能集成良好
- **给定**: 包含文本识别动作的脚本
- **当**: 执行该脚本
- **则**: 脚本整体运行流畅，文本识别结果可被后续动作通过变量引用使用
- **验证方式**: programmatic

## 开放问题

- 是否需要支持多语言配置？（初始阶段使用自动检测）
- 是否需要支持正则表达式匹配文本？（初始阶段只支持精确包含匹配）
