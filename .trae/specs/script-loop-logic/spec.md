# 脚本循环逻辑实现 - 产品需求文档

## 概述
- **摘要**：在 AutomationEngine 中实现脚本操作的循环执行逻辑，支持有限次数循环和无限循环
- **目的**：解决目前 LoopStart 和 LoopEnd 动作只有占位符，无法实际执行循环的问题
- **目标用户**：使用 AutoBot RPA 创建自动化脚本的用户

## 目标
- 实现 LoopStart 和 LoopEnd 动作的完整执行逻辑
- 支持有限次数循环（用户指定次数）
- 支持无限循环（直到用户手动停止）
- 正确处理循环嵌套（可选，作为增强功能）
- 在执行日志中清晰显示循环执行状态

## 非目标（不包含范围）
- 不实现条件循环（如 while 循环），仅支持基于 LoopStart 和 LoopEnd 的固定次数循环
- 不实现循环中断（break）和继续（continue）功能

## 背景与上下文
- 目前项目已在 Script.kt 中定义了 LoopStart 和 LoopEnd 数据类
- AutomationEngine.executeActions() 中有循环动作的占位处理，但没有实际逻辑
- 条件分支功能已经实现，可供参考
- 项目采用 Kotlin + Jetpack Compose + Room 架构

## 功能需求
- **FR-1**：检测 LoopStart 和 LoopEnd 的配对关系
- **FR-2**：执行 LoopStart 和 LoopEnd 之间的动作指定次数
- **FR-3**：支持无限循环（infinite = true）
- **FR-4**：在执行日志中显示循环执行次数和状态
- **FR-5**：处理循环动作的暂停和恢复
- **FR-6**：处理循环动作的停止

## 非功能需求
- **NFR-1**：循环执行性能无明显延迟
- **NFR-2**：循环逻辑代码易于维护和扩展
- **NFR-3**：正确处理错误情况，如未配对的 LoopStart/LoopEnd

## 约束
- **技术**：必须使用 Kotlin 协程实现，符合现有代码风格
- **业务**：需要保持与现有条件分支实现风格一致
- **依赖**：仅使用项目现有依赖，不引入新库

## 假设
- 脚本中的 LoopStart 和 LoopEnd 是正确配对的（虽然需要处理错误情况）
- 用户会按照正确顺序添加 LoopStart 和 LoopEnd 动作
- 循环逻辑可以通过栈或配对检测实现

## 接受标准

### AC-1：循环配对检测
- **给定**：脚本包含 LoopStart 和 LoopEnd 动作
- **当**：执行脚本时
- **则**：系统能够正确检测 LoopStart 对应的 LoopEnd 位置
- **验证**：programmatic
- **备注**：如果找不到配对的 LoopEnd，应记录错误日志并跳过循环

### AC-2：有限次数循环执行
- **给定**：脚本包含 LoopStart（times = N）和 LoopEnd 动作，中间有其他动作
- **当**：执行脚本时
- **则**：LoopStart 和 LoopEnd 之间的动作会被准确执行 N 次
- **验证**：programmatic

### AC-3：无限循环执行
- **给定**：脚本包含 LoopStart（infinite = true）和 LoopEnd 动作
- **当**：执行脚本时
- **则**：LoopStart 和 LoopEnd 之间的动作会被不断重复执行，直到用户手动停止
- **验证**：programmatic

### AC-4：循环执行日志
- **给定**：脚本正在执行循环
- **当**：每次循环开始和结束时
- **则**：执行日志中会显示当前循环次数和状态
- **验证**：programmatic

### AC-5：循环暂停和恢复
- **给定**：脚本正在执行循环
- **当**：用户暂停执行后再恢复
- **则**：循环会从暂停处继续执行，保持当前循环次数
- **验证**：programmatic

### AC-6：循环停止
- **给定**：脚本正在执行循环
- **当**：用户停止执行
- **则**：循环立即停止，不再继续执行后续迭代
- **验证**：programmatic
